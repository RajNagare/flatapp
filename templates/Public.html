GUS

The web app kickstarter

Throw me a bone: why do we need Gus?

Building web apps sucks and we needed something better. We wanted an app that was easy to read, fun to write, and provided the best performance.

  • PHP driven (as in the easiest way possible)
  • Templated with Twig
  • Route-controller based
  • Has All the whiz-bang js + css libs out there like jive, bootstrap, and modernizr

Treaching an old dog new tricks

Gus is simple: it's all based off of the route/controller practice. You have URLS or routes like /about or /dashabord ,that you want specfic logic or controllers behind them.

By writing a controller and placing it in the /controller directory, gus automatically starts routing to that controller. If a template is named the same as the controller, it will be the route-specific template displayed.

What does this mean?

Sally wants users to go to /groceries and see a list of groceries. She wants to set up the /groceries route. She would write the logic for that route @ controller/groceries.php . Hint hint, look at the route and the controller name.

Okay, so then how do we use templates?

Sally needs to display the items in a template. She writes a template named the same exact thing in /templates, called groceries.html. This is used as the route template and it always extends a global.html template

Libraries!

The /lib is where all 3rd party libraries go. If you want to integrate things like database object interaction, mailer wrappers, or CSS frameworks, this is the place to do it.

But what if I want to do xyz

Gus wants you to have freedom. This is just a jump of point for web applications, so if you want to do xyz, then skies the limit.

Install that puppy

Git It Bro! and then:

  • Update config.ini in the root directory to match your server's absolute path for the application
  • Make sure that templates/cache/ is writeable by apache (or the like)
  • Make sure .htaccess is in the root directory and that apache is correctly configured to use mod_rewrites
  • Rage n3wbs
